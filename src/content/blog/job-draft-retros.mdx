本職はソフトウェアエンジニアをしています。ここ最近はエンジニアリングマネージャー業も増えてきました。そんな僕が転職ドラフトを久しぶりに受けてみたので、結果をできるだけ開示しつつ今後のキャリアについて書いてみようと思います。年収とかもざっくり書くので結構生々しいかもしれません。

## 転職ドラフトを受けた際の自分のスペック

- ネットワークエンジニア 半年
  - CCNA, LPIC1,2
- Web開発の実務経験 7+年（フルスタック）
  - インフラレイヤの経験浅め。terraformなどの構成管理ツールでインフラ構築は可能
- 経験技術
  - サーバーサイド: Ruby, Ruby on Rails, Go, Java, Python, Node.js
  - フロントエンド: JavaScript, TypeScript, React, Redux, Next.js, Vite, HTML/CSS
  - ミドルウェア: Redis, Sendgrid, MySQL, PostgreSQL, Unicorn, Puma, Nginx
  - その他: Linux, Shell, Datadog, NewRelic, Zabix, Docker, gRPC, CircleCI, GitHub Actions, Kubernetes, AWS, GCP, Selenium
- 経験業務
  - 基本設計、詳細設計、コーディング、ユニットテスト、単体テスト、運用保守、ドキュメント作成、障害クライテリア作成、モデリング、DB設計、パフォーマンス・チューニング、コードレビュー、採用業務（カジュアル、一次面接、技術面接）、プロジェクトマネジメント、API設計
  - 小規模のチーム（～6人）のマネジメント経験有り
- テックリード4年
- エンジニアリングマネージャー1年
- 年収約800万

レジュメですが、書くべき内容とフォーマットを決めて下書きした上で、ChatGPTに推敲を丸投げしました。その上で転職ドラフトのレジュメチェックを通して良いレジュメのお墨付きをもらっています。

## なんで転職ドラフトを受けたの？

動機の部分では多くの人が気になる所だと思います。

正直に言うと、**業務内容や任される責任に対して支払われるフィーが適当なのか分からなくなった**ことが一番でした。

その他の理由を箇条書きにすると下記のような感じです。

- 現社の勤続年数が長くなっており、積んできたナレッジで上手くやってる感覚があった
- 業務委託（個人事業主）の稼働が減ったため、市場評価に対する意識が鈍化していた
- 社内異動制度などを利用して部署異動を何度かしたが、1年もするとコンフォートゾーンに入るため、環境を変えることに興味があった

一方で、ここ2年ぐらいは働く環境がガラッと変わりました。具体的にはメインで利用する言語が日本語から英語になりました。同じチームで働く同僚は、現チームでは僕以外外国籍の方です。非常に良い経験ができていると思います。
環境には非常に感謝しています。また拙い英語で、マネジメントという慣れない業務を行うのは非常に難易度が高いですが、グローバルな環境でチームとして成果を生むというのは非常にエキサイティングで面白いことだなと考えるようになりました。

**将来的には米国のテックカンパニーにチャレンジしたいなという気持ちもある**ので、もう1社ぐらい内資企業で経験を積みたいとも考えていました。

## その後と所感

転職については結構前向きに考えています。
しかし、転職ドラフト開催後の年俸改定でRSや賞与を含めると悩ましい金額になってしまいました。

現時点では何社かとカジュアル面談させてもらい、琴線に触れる会社があれば転職したいなと思っています。
一方、近い将来外資ITにチャレンジすることを考えると、今の環境もかなり良いなと何度かの面談を経て再確認もしています。なので、じっくり考えようと思います。

今回転職を改めて考えてみて、大きな学びがありました。

- 他社からの見え方で自分が今後どうなりたいかのアウトラインが考えられる
- 市場で評価されるにはという意識ができると、業務や業務外においてよりモチベートできる
